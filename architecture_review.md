# Kizuna Executive Engineering Summary

| Domain | Core Issue (Con) | Recommended Solution | Implementation Downsides | Estimated Cost |
| :--- | :--- | :--- | :--- | :--- |
| **📡 WebRTC & P2P Architecture** | **"Full Mesh" Scalability Limit.** In a 20-person room, there are 190 peer connections. This exponential growth melts mobile CPUs and saturates bandwidth. | Implement a **Star Topology** (elect a "Super Peer" router) or use an external **SFU** (Selective Forwarding Unit) server. | SFUs remove true P2P decentralization. "Super Peer" topologies are highly fragile (if the peer's internet drops, the network collapses until a new one is elected). | **High.** SFU servers charge per minute/GB. Super Peers require significant upfront R&D time. |
| **🔒 Cybersecurity** | **IP Leaks & Payload Spoofing.** Direct P2P exposes public IPs to all participants. Clients also implicitly trust incoming WebRTC payloads, allowing message spoofing. | Force all connections through **TURN Relays** to mask IPs. Implement **Cryptographic Signatures** (e.g., `libsodium`) for sender identity verification. | Forcing TURN relays adds latency. Cryptographic signatures increase payload sizes and drastically complicate initial handshakes. | **Medium-High.** TURN servers incur steep recurring bandwidth bills. Cryptography is a pure upfront engineering cost. |
| **🎨 UI/UX Performance** | **DOM Bloat.** Chat history appends infinitely to the DOM. Thousands of messages will bloat browser memory, causing scrolling to stutter and the page to freeze. | **UI Virtualization.** Implement a "virtualized list" (windowing) that only renders the ~20 messages currently visible on the screen. | Extremely difficult to implement for elements with dynamic heights (like images). It entirely breaks the browser's native `Ctrl+F` search functionality. | **Low.** Zero recurring infrastructure costs. Requires ~1-2 weeks of upfront senior frontend developer time. |
| **🏗️ Messaging State** | **The "Split Brain" Problem.** If the network splits, users develop diverging histories. Basic array-merging upon reconnection overwrites edits and loses reactions. | Swap the array-based chat state for a **CRDT** (Conflict-free Replicated Data Type) like *Yjs* or *Automerge* to guarantee mathematical synchronization. | Significantly increases the JS bundle size. Requires a fundamental rewrite of the entire core messaging engine and custom Supabase snapshot logic. | **High (R&D).** No recurring monthly cost, but the massive rewrite requires roughly a month of specialized developer time. |
